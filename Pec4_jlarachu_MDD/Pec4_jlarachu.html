1. Combinación de clasificadores (7.5 puntos)¶

El ensemble learning es una estrategia en la que se utiliza un grupo de modelos para resolver un problema mediante la combinacin estratgica de diversos modelos de aprendizaje automtico en un slo modelo predictivo.

En general, los mtodos de ensemble se utilizan principalmente para mejorar la precisin del rendimiento general de un modelo y combinar varios modelos diferentes, tambin conocidos como aprendices bsicos, para predecir los resultados, en lugar de utilizar un slo modelo.

Por qu entrenamos tantos clasificadores diferentes en lugar de uno solo? Bueno, el uso de varios modelos para predecir el resultado final en realidad reduce la probabilidad de sopesar las decisiones tomadas por modelos deficientes (sobreentrenados, no debidamente ajustados...).

Cuanto ms diversos sean estos aprendices bsicos, ms poderoso ser el modelo final.

Tengamos en cuenta que en cualquier modelo de aprendizaje automtico, el error de generalizacin viene dado por la suma de cuadrados de bias + varianza + error irreductible.

Los errores irreductibles son algo que est ms all de nosotros! No podemos reducirlos.

Sin embargo, utilizando ensembles, podemos reducir el sesgo (bias) y la varianza de un modelo. Esto reduce el error de generalizacin general.

La compensacin de sesgo-varianza es el punto de referencia ms importante que diferencia un modelo robusto de uno inferior (entendamos por inferior un modelo no demasiado generalizable).

Aunque no es una regla exacta, en el aprendizaje automtico, los modelos que tienen un sesgo alto tienden a tener una varianza ms baja y viceversa.

Hemos estado hablando de bias y varianza. Pero veamos que entendemos por sesgo de un modelo y por varianza de un modelo.

  1. Sesgo: El sesgo es un error que surge debido a suposiciones falsas realizadas en la fase de aprendizaje de un modelo. Un sesgo alto puede hacer que un algoritmo de aprendizaje omita informacin importante y correlaciones entre las variables independientes y las etiquetas de clase, por lo que no se ajusta al modelo.

  2. Varianza: la varianza nos dice qu tan sensible es un modelo a los pequeos cambios en los datos de entrenamiento. Es decir, cunto cambia el modelo. Una gran variacin en un modelo lo har propenso al ruido aleatorio presente en el conjunto de datos, por lo que se ajustar demasiado al modelo.

Para comprender con ms detalle la compensacin de sesgo y varianza en los modelos de aprendizaje automtico, podes consultar este artculo.

Una vez llegados a este punto, podemos dividir los ensembles en cuatro categoras:

  1. Bagging: El bagging se utiliza principalmente para reducir la variacin en un modelo. Un ejemplo simple de bagging es el algoritmo Random Forest.

  2. Boosting: El boosting se utiliza principalmente para reducir el sesgo en un modelo. Ejemplos de algoritmos de impulso son Ada-Boost, XGBoost, rboles de decisin mejorados por gradiente, etc.

  3. Stacking: el stacking se utiliza principalmente para aumentar la precisin de prediccin de un modelo.

  4. Cascading: esta clase de modelos son muy precisos. La conexin en cascada se usa principalmente en escenarios en los que no puede permitirse cometer un error. Por ejemplo, una tcnica en cascada se usa principalmente para detectar transacciones fraudulentas con tarjetas de crdito.

El ensemble learning es una estrategia sofisticada que implica la utilizacin de mltiples modelos para abordar un problema mediante la combinacin estratgica de diversos modelos de aprendizaje automtico en un modelo predictivo consolidado.

Los errores irreductibles son inherentemente incontrolables, pero los mtodos de conjunto, mediante la compensacin de sesgo y varianza, tienen como objetivo reducir tanto el sesgo como la varianza para mejorar el rendimiento global del modelo.

1.2.2. Bagging¶

La idea central del bagging es usar rplicas del conjunto de datos original y usarlas para entrenar diferentes clasificadores.

Crearemos subconjuntos muestreando aleatoriamente un montn de puntos del conjunto de datos de entrenamiento con reemplazo.

Ahora entrenaremos clasificadores individuales en cada uno de estos subconjuntos bootstrap.

Cada uno de estos clasificadores base predecir la etiqueta de clase para un problema dado. Aqu es donde combinamos las predicciones de todos los modelos base. Esta parte se llama etapa de agregacin. Es por eso que encontraris los ensembles bagging por el nombre de ensembles de agregacin.

Por lo general, se usa un voto de mayora simple en un sistema de clasificacin y se toma la media de todas las predicciones para los modelos de regresin para combinar todos los clasificadores base en un solo modelo y proporcionar el resultado final del modelo de conjunto.

Un ejemplo simple de tal enfoque es el algoritmo Random Forest. El bagging reduce la alta variacin (varianza) de un modelo, reduciendo as el error de generalizacin. Es un mtodo muy eficaz, especialmente cuando tenemos datos muy limitados con pudiera ser nuestro caso.

Mediante el uso de muestras de bootstrap, podemos obtener una estimacin agregando las puntuaciones de muchas muestras.

Cmo haramos bagging?

Supongamos que tenemos un conjunto de entrenamiento que contiene 100.000 puntos de datos.

Crearamos N subconjuntos muestreando al azar 50K puntos de datos para cada subconjunto.

Cada uno de estos N subconjuntos se utilizar para entrenar N clasificadores diferentes.

En la etapa de agregacin, todas estas N predicciones se combinarn en un solo modelo, tambin llamado metaclasificador.

De los 100.000 puntos presentes originalmente en el conjunto de datos, si eliminamos 1000 puntos, el impacto que tendr en los conjuntos de datos muestreados ser muy inferior.

Si pensamos intuitivamente, es posible que algunos de estos 1000 puntos no estn presentes en todos los conjuntos de datos muestreados y, por lo tanto, la cantidad de puntos que se eliminarn de cada conjunto de datos muestreados ser muy inferior. Incluso cero en algunos casos! En resumen, el impacto de eliminar 1000 puntos de este tipo ser en realidad menor en los clasificadores base, lo que reducir la variacin en un modelo y lo har ms slido.

La varianza no es ms que sensibilidad al ruido, como hemos comentado anteriormente.

1.2.3. Boosting¶

El boosting se utiliza para convertir a los clasificadores de base dbil en fuertes. Los clasificadores dbiles generalmente tienen una correlacin muy dbil con las etiquetas de clase verdaderas y los clasificadores fuertes tienen una correlacin muy alta entre el modelo y las etiquetas de clase verdaderas.

El boosting capacita a los clasificadores dbiles de manera iterativa, cada uno tratando de corregir el error cometido por el modelo anterior. Esto se logra entrenando un modelo dbil en todos los datos de entrenamiento, luego construyendo un segundo modelo que tiene como objetivo corregir los errores cometidos por el primer modelo. Luego construimos un tercer modelo que intenta corregir los errores cometidos por el segundo modelo y as sucesivamente. Los modelos se agregan de forma iterativa hasta que el modelo final ha corregido todos los errores cometidos por todos los modelos anteriores.

Cuando se agregan los modelos en cada etapa, se asignan algunos pesos al modelo que est relacionado con la precisin del modelo anterior. Despus de agregar un clasificador dbil, los pesos se vuelven a ajustar. Los puntos clasificados incorrectamente reciben pesos ms altos y los puntos clasificados correctamente reciben pesos ms bajos. Este enfoque har que el siguiente clasificador se centre en los errores cometidos por el modelo anterior.

El boosting reduce el error de generalizacin tomando un modelo de alto bias y baja varianza y reduciendo el bias en un nivel significativo. Recuerde, el bagging reduce la varianza. Al igual que el bagging, el boosting tambin nos permite trabajar con modelos de clasificacin y regresin.

1.3.3. Stacking¶

Todos los modelos individuales se entrenan por separado en el conjunto completo de datos de entrenamiento y se ajustan para lograr una mayor precisin. La compensacin de bias y varianza se tiene en cuenta para cada modelo. El modelo final, tambin conocido como metaclasificador, se alimenta de las etiquetas de clase predichas por los modelos base o de las probabilidades predichas para cada etiqueta de clase. Luego, el metaclasificador se entrena en funcin de los resultados dados por los modelos base.

En el stacking, se entrena un nuevo modelo en funcin de las predicciones realizadas por los modelos anteriores. Este proceso se lleva a cabo de forma secuencial. Esto significa que varios modelos se entrenan en la etapa 1 y se ajustan con precisin. Las probabilidades pronosticadas de cada modelo de la etapa 1 se alimentan como entrada a todos los modelos en la etapa 2. Los modelos en la etapa 2 luego se ajustan con precisin y las salidas correspondientes se alimentan a los modelos en la etapa 3 y as sucesivamente. Este proceso se produce varias veces en funcin de la cantidad de capas de apilamiento que desee utilizar.

La etapa final consiste en un nico modelo que nos da el resultado final al combinar el resultado de todos los modelos presentes en las capas anteriores.

A menudo, el uso de clasificadores apilables aumenta la precisin de prediccin de un modelo. Pero de ninguna manera puede garantizarse que el uso de apilamiento aumente la precisin de la prediccin en todo momento! Echad un vistazo al siguiente link.

2.4. Modelo de red neuronal paralela (0.75 ptos)¶

Cree y entrene un modelo de red neuronal paralela al que llamar model_par.

En este modelo, debe definir dos rutas de entrada separadas que eventualmente se fusionarn antes de conducir a la capa de salida. Este tipo de arquitectura puede resultar til en casos en los que su modelo necesita aprender de dos conjuntos diferentes de caractersticas o representaciones de datos.

Defina dos rutas de entrada:¶

input_a y input_b se definirn como dos entradas separadas. Ambos esperan datos de forma (X_train.shape[1],), que es la cantidad de funciones en su conjunto de datos.

Capas paralelas:¶

path_a y path_b representan dos capas (o rutas) paralelas en la red. Cada uno procesar su entrada de forma independiente. Ambas rutas deben tener una capa densa con 64 neuronas y utilizar la funcin de activacin ReLU.

Concatenación de rutas:¶

La capa de concatenacin fusionar las salidas de path_a y path_b. Esto permitir que el modelo aprenda de ambas vas simultneamente.

Capa de salida:¶

La capa de salida debe ser una capa densa con 3 neuronas y una funcin de activacin softmax, adecuada para clasificacin de clases mltiples.

Construyendo y compilando el modelo:¶

El modelo (model_par) debe definirse con dos entradas y una salida. Se compilar con el optimizador Adam y la funcin de prdida categorical_crossentropy, que son estndar para tareas de clasificacin.

Entrenando el modelo:¶

El modelo debe entrenarse en X_train para ambas entradas ([X_train, X_train]). Esto implicar que est alimentando los mismos datos a travs de ambas rutas. Las etiquetas de destino (y_train) deben usarse para el entrenamiento. El modelo se entrenar durante 50 pocas (epochs) con un tamao de lote de 10 (batch_size). El parmetro detallado = 1 (verbose) en ajuste mostrar una barra de progreso durante el entrenamiento, lo que puede ser til para monitorear el proceso de entrenamiento.

def plot_roc_curve(y_true, y_pred_prob, title):
    n_classes = y_true.shape[1]

    fpr = dict()
    tpr = dict()
    roc_auc = dict()

    for i in range(n_classes):
        fpr[i], tpr[i], _ = roc_curve(y_true[:, i], y_pred_prob[:, i])

    fpr["macro"], tpr["macro"], _ = roc_curve(y_true.ravel(), y_pred_prob.ravel())
